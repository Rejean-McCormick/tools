Voici un **plan d’update complet pour l’app Smart Dumper**.

## Objectif

Ajouter un mode **AI Navigation** qui améliore les dumps pour ChatGPT :

```text
- références de lignes
- références de chunks
- index des classes/fonctions
- index des imports
- résumés courts par fichier
- patch targets
- optionnel : lignes physiquement numérotées
```

L’index actuel ne donne que `path -> volume`, donc l’amélioration principale consiste à enrichir ce master index avec des métadonnées plus précises.  Les volumes ont déjà `lines` et `chunks`, donc on peut étendre ce format sans casser la structure existante.

## Phase 3 — Enrichir les métadonnées par fichier

### Fichier à modifier

```text
worker/file_processing.py
```

### Pourquoi

`FileProcessor.process_file_content()` retourne déjà les métadonnées principales : `rel_path`, `ext`, `size_bytes`, `line_count`, `kind`, `file_id`, `chunks`, `content`. C’est donc le bon endroit pour ajouter `symbols`, `imports`, `summary`, `line_ref`, `chunk_refs`. 

### Modifs

Ajouter à la dataclass :

```python
ai_line_references: bool = True
number_source_lines: bool = False
symbol_index: bool = True
import_index: bool = True
file_summaries: bool = True
line_number_width: int = 6
```

Ajouter dans le dict retourné :

```python
"line_ref": "1-526",
"chunk_refs": "1-350,351-526",
"symbols": [...],
"imports": [...],
"summary": "...",
"numbered_content": "...",
```

### Règle importante

Ne pas remplacer `content` par `numbered_content`.

Garder :

```python
content = code propre
numbered_content = version annotée
```

Comme ça, le dump reste copiable sans numéros si nécessaire.

## Phase 4 — Enrichir les volumes texte

### Fichier à modifier

```text
worker/writers_text.py
```

### Pourquoi

Ce fichier écrit les volumes texte, incluant `FILE_INDEX`, `FILE BEGIN`, les chunks et le contenu. 

### Modifs dans `FILE_INDEX`

Passer de :

```text
ENTRY id=... path="worker/dump_worker.py" kind=source size=21482 lines=526 chunks=2
```

à :

```text
ENTRY id=... path="worker/dump_worker.py" kind=source size=21482 lines=526 line_ref=1-526 chunks=2 chunk_refs=1-350,351-526 symbols=12 imports=9 summary="Orchestrates repository dumps"
```

### Modifs dans chaque `FILE BEGIN`

Ajouter :

```text
line_ref=1-526
chunk_refs=1-350,351-526
summary="Orchestrates repository dumps"
```

Puis :

```text
--- SYMBOLS ---
class DumpWorker line=27 end_line=526
def run line=279 end_line=526

--- IMPORTS ---
import os line=4
from pathlib import Path line=6
from .index import IndexWriterMixin line=18
```

### Modif du return

Retourner un nouveau champ :

```python
"file_entries": [...]
```

Garder aussi :

```python
"contained_files": [...]
```

pour compatibilité avec l’ancien code.

## Phase 6 — Enrichir le master index

### Fichier à modifier

```text
worker/index.py
```

### Pourquoi

L’index actuel contient les volumes et une section `FILE LOCATOR (path -> volume)`, mais pas les lignes, chunks, symboles ou imports. 

### Garder l’ancien format

Conserver :

```text
==== FILE LOCATOR (path -> volume) ====
worker/dump_worker.py    volume=smart_dumper_..._02_worker.txt
```

### Ajouter `FILE DETAIL INDEX`

```text
==== FILE DETAIL INDEX ====
worker/dump_worker.py
  volume=smart_dumper_20260518_144408_02_worker.txt
  id=5d8b3d641544
  kind=source
  lines=526
  line_ref=1-526
  chunk_refs=1-350,351-526
  symbols=12
  imports=9
  summary=Orchestrates repository dumps
```

### Ajouter `SYMBOL INDEX`

```text
==== SYMBOL INDEX ====
class DumpWorker    path=worker/dump_worker.py    volume=...    line=27    end_line=526
def run             path=worker/dump_worker.py    volume=...    line=279   end_line=526
```

### Ajouter `IMPORT INDEX`

```text
==== IMPORT INDEX ====
worker/dump_worker.py    from pathlib import Path    line=6
worker/dump_worker.py    from .index import IndexWriterMixin    line=18
```

### Ajouter `PATCH TARGETS`

```text
==== PATCH TARGETS ====
GUI options                  path=gui.py
Worker orchestration         path=worker/dump_worker.py
Per-file metadata            path=worker/file_processing.py
Text output format           path=worker/writers_text.py
XML output format            path=worker/writers_xml.py
Master index format          path=worker/index.py
Generated AI instructions    path=worker/instructions.py
Defaults/constants           path=constants.py
```

### Ajouter dans le fichier généré

```markdown
## AI navigation features

- Use `FILE DETAIL INDEX` to find exact file ranges.
- Use `SYMBOL INDEX` to locate classes/functions.
- Use `IMPORT INDEX` before expanding dependencies.
- Use `line_ref` and `chunk_refs` to read only the required range.
- Use `PATCH TARGETS` to identify likely files to modify.
```

# Critères d’acceptation

L’update est réussie si un dump généré permet de répondre vite à ces questions sans lire tout le repo :

```text
Où est la classe DumpWorker?
Quels fichiers importent worker/index.py?
Quelles fonctions sont dans worker/file_processing.py?
Quel chunk contient DumpWorker.run?
Quels fichiers modifier pour changer le format du master index?
```

Le résultat attendu : **moins de lecture inutile, citations plus précises, meilleure navigation, et patchs plus rapides**.
